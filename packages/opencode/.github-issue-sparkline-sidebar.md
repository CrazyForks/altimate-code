# GitHub Issue: Add sparkline cost/usage trends in sidebar

**Title:** Add sparkline cost/usage trends in sidebar

## Summary

Add tiny sparkline charts (▁▂▃▅▇) next to cost and token metrics in the sidebar showing trends over the session.

## Motivation

Data engineers are metrics-obsessed. Seeing `Cost: $0.42 ▁▂▃▅▇▃▂` instantly communicates "that last query was expensive" without needing a separate dashboard. The [Smashing Magazine dashboard UX guide](https://www.smashingmagazine.com/2025/09/ux-strategies-real-time-dashboards/) specifically recommends pairing sparklines with directional indicators like `▲ +3.2%`.

## Proposed Implementation

### Sparkline Types
1. **Token usage sparkline** per message — shows token consumption trend across the session
2. **Cost-per-tool-call sparkline** — highlights expensive operations
3. **Warehouse query cost sparkline** (from FinOps tools) — tracks warehouse spend

### Technical Details
- Use Unicode block characters: `▁▂▃▄▅▆▇█` (U+2581 through U+2588)
- Add sparkline rendering utility in `packages/opencode/src/cli/cmd/tui/util/sparkline-utils.ts`
- Integrate into `sidebar.tsx` Context section, next to cost and token displays
- Track per-message cost/token data in a rolling window (last N messages)
- Color-code: green for below average, yellow for above, red for 2x+ average
- Respect `animations_enabled` setting — show static last value when animations disabled
- Ensure graceful degradation when terminal doesn't support Unicode block chars

### Sidebar Mockup
```
Context
12,345 tokens
42% used
$0.42 spent ▁▂▃▅▇▃▂
```

### Design Considerations
- Sparkline width should be fixed (7-10 chars) regardless of terminal width
- Use directional indicator: `▲` / `▼` with percentage delta from previous
- Dim the sparkline when no data points yet
- Pair with existing theme colors (success for low cost, warning for high)

## Impact
- **Effort**: Low (pure utility + sidebar integration)
- **Impact**: Medium — gives data engineers instant cost intuition
- **Data-Team Appeal**: ★★★★☆
